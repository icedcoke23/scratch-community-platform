# 🔧 技术栈选型

> 日期：2026-04-23

---

## 一、整体选型

| 层级 | 推荐技术栈 | 理由 |
|---|---|---|
| **前端框架** | Vue 3 + TypeScript + Vite | 现代化、生态好、学习曲线适中 |
| **UI 组件库** | Ant Design Vue 4.x / Naive UI | 成熟稳定、组件丰富 |
| **Scratch 编辑器** | scratch-vm + scratch-gui (魔改) | 官方核心，社区生态好 |
| **代码编辑器** | Monaco Editor | VS Code 同款引擎 |
| **Python 运行** | Pyodide (浏览器端) / 后端沙箱 | 双端支持 |
| **API 网关** | Kong / APISIX | 高性能、插件丰富 |
| **后端框架** | Spring Boot 3.x 或 Go (Gin/Fiber) | Java 生态成熟 / Go 高性能 |
| **ORM** | MyBatis-Plus / GORM | 按语言选 |
| **数据库** | MySQL 8.0 (核心) + MongoDB (作品JSON) | 关系型 + 文档型互补 |
| **缓存** | Redis 7.x (Cluster) | 积分/排行榜/实时数据 |
| **搜索引擎** | Elasticsearch 8.x | 全文搜索/标签筛选 |
| **消息队列** | RocketMQ / RabbitMQ | 判题任务异步处理 |
| **文件存储** | MinIO (自建) / 阿里云 OSS | 作品/素材/视频 |
| **容器编排** | Docker + Kubernetes | 微服务部署/弹性伸缩 |
| **CI/CD** | GitLab CI / GitHub Actions | 自动化部署 |
| **监控** | Prometheus + Grafana | 全链路监控 |
| **AI 接入** | OpenAI API / 本地 LLM | 通用 AI 能力 |
| **判题沙箱** | Docker 容器隔离 | 安全执行用户代码 |

## 二、前端框架对比

| 维度 | Vue 3 | React | 选择 |
|---|---|---|---|
| 学习曲线 | 中等 | 较陡 | Vue 3 ✅ |
| 生态成熟度 | 高 | 很高 | 平手 |
| TypeScript 支持 | 好 | 很好 | 平手 |
| 团队熟悉度 | 高 | — | Vue 3 ✅ |
| Scratch 集成 | 方便 | 方便 | 平手 |
| 构建工具 | Vite (极快) | CRA/Vite | Vue 3 ✅ |

**结论**：选择 Vue 3 + TypeScript + Vite

## 三、后端框架对比

| 维度 | Spring Boot 3 | Go (Gin) | 选择 |
|---|---|---|---|
| 生态成熟度 | 极高 | 高 | Spring Boot ✅ |
| 微服务支持 | Spring Cloud | go-micro/kratos | Spring Boot ✅ |
| 性能 | 高 | 极高 | Go ✅ |
| 开发效率 | 高 | 中等 | Spring Boot ✅ |
| 人才市场 | 丰富 | 较丰富 | Spring Boot ✅ |
| 判题沙箱集成 | Docker SDK | Docker SDK | 平手 |

**结论**：主力 Spring Boot 3，判题引擎可用 Go 重写以获取更高性能

## 四、数据库选型

| 数据库 | 用途 | 理由 |

| MySQL 8.0 | 核心业务数据 | ACID 事务、成熟稳定 |

## 五、Scratch 判题引擎技术方案

### 方案对比

| 方案 | 原理 | 优点 | 缺点 |
|---|---|---|---|
| A: 积木→代码转换 | 解析积木 JSON → 生成 Python/JS → 执行 | 可精确控制、支持复杂逻辑 | 转换复杂、需维护转换器 |
| B: VM Headless 运行 | 在 Node.js 中运行 scratch-vm → 注入输入 → 捕获输出 | 最接近真实运行环境 | 性能开销大、需修改 VM |
| C: 舞台截图比对 | 运行项目 → 截取舞台 → 与标准截图比对 | 简单直观 | 不精确、不支持算法题 |
| D: 混合方案 | 算法题用方案 A/B，创意题用方案 C + AI | 灵活、覆盖面广 | 复杂度高 |

**结论**：选择方案 D（混合方案），算法题优先方案 B，创意题用 AI 评分

## 七、部署架构

### 开发环境
```
docker-compose.yml:
  - MySQL 8.0
  - Redis 7
  - MongoDB 6
  - MinIO
  - 各微服务容器
```

### 生产环境
```
Kubernetes 集群:
  - Ingress (Nginx)
  - 各微服务 Deployment + HPA
  - MySQL 主从
  - Redis Cluster
  - MongoDB ReplicaSet
  - MinIO 分布式
  - Prometheus + Grafana 监控
```
