## sb3 解析引擎技术方案

### 核心原理

```
.sb3 文件 = ZIP 压缩包
  ├── project.json          ← 核心：所有代码/角色/变量/积木
  ├── abc123.png            ← 造型图片
  ├── def456.wav            ← 声音文件
  └── ...

解析步骤:
  1. ZIP 解压 → 得到 project.json + 素材文件
  2. JSON 解析 → 提取 targets(舞台+角色)、blocks、variables、lists、broadcasts
  3. 遍历 blocks → 构建程序链(从 event_whenflagclicked 等帽子积木开始)
  4. 统计分析 → 积木类型分布/嵌套深度/复杂度评分
  5. 输出结构化 JSON → 存入 MongoDB
```

### 关键数据结构 (project.json)

```json
{
  "targets": [
    {
      "isStage": true,
      "name": "Stage",
      "variables": {"varId1": ["分数", 0]},
      "lists": {"listId1": ["排行榜", []]},
      "broadcasts": {"bcastId1": "开始游戏"},
      "blocks": {...},        // 积木数据
      "costumes": [...],      // 背景造型
      "sounds": [...]
    },
    {
      "isStage": false,
      "name": "小猫",
      "variables": {...},
      "blocks": {...},        // 该角色的积木
      "costumes": [...],      // 角色造型
      "sounds": [...],
      "x": 0, "y": 0,
      "size": 100,
      "direction": 90,
      "visible": true
    }
  ],
  "monitors": [...],          // 变量监视器
  "extensions": [...]         // 使用的扩展
}
```

### 实现技术

| 方案 | 选择 |
|---|---|
| 解压 | Node.js: adm-zip / JSZip |
| JSON 解析 | 原生 JSON.parse |
| 积木链构建 | 自研：遍历 blocks → 以 opcode 为 key 构建链 |
| VM Headless 运行 | scratch-vm (Node.js) |
| 素材提取 | 直接读取解压后的文件 |
| 复杂度评分 | 自研算法：积木类型加权 + 嵌套深度 + 程序链数 |

## 部署架构

### 开发环境
```yaml
# docker-compose.yml
services:
  mysql:    # MySQL 8.0
  redis:    # Redis 7
  mongodb:  # MongoDB 6
  minio:    # MinIO (文件存储)
  api:      # Spring Boot 后端
  web:      # Vue 3 前端 (Nginx)
```

### 生产环境
```
Nginx → Vue 3 静态文件
      → /api → Spring Boot (多实例)
      → /ws  → WebSocket (云变量)
```
