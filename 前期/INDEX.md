## 📌 决策记录

| 日期 | 决策 | 理由 |
|---|---|---|
| 2026-04-23 | 聚焦 Scratch，暂不扩展 | 降低 MVP 复杂度 |
| 2026-04-23 | 不做工作室/组队创作 | 简化社区 |
| 2026-04-23 | 教学面向课后 | 作业/测评/班级管理 |
| 2026-04-23 | sb3 解析是核心差异化 | 自动解压→解析→AI/教师分析 |
| 2026-04-23 | 架构审计：微服务→模块化单体 | 13 服务过度拆分 |
| 2026-04-23 | sb3-parser 从服务改为共享库 | 纯计算逻辑 |
| 2026-04-23 | MVP 存储简化：去掉 MongoDB/ES | Phase 1-2 仅 MySQL+Redis+MinIO |
| 2026-04-23 | **v0.5 深度优化：14 模块→6 模块** | 真正可交付的 MVP，6-8 周 2-3 人可完成 |
| 2026-04-23 | **AI Gateway 从独立服务改为 SDK 直连** | MVP 阶段 AI 调用量小，不值得多一个服务 |
| 2026-04-23 | **部署单元从 3+2 简化为 2 个** | 去掉 AI Gateway，合并进主服务 |
| 2026-04-23 | **数据库从 40 张表砍到 18 张** | 只保留 MVP 必需表，28 张表延后 |
| 2026-04-23 | **防刷引擎简化为 Redis 计数器** | 7 条规则→3 条简单规则 |
| 2026-04-23 | **内容审核简化为敏感词匹配** | 三级流水线→单级 |
| 2026-04-23 | **判题 Level 3 (浏览器沙箱) 延后到 Phase 3** | 资源消耗大，MVP 不需要 |
| 2026-04-23 | **竞赛/推荐/成长路径/教师高级功能 全部延后** | 聚焦核心创作+判题循环 |
