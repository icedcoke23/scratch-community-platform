## 一、产品定位

**一句话**：面向 K12 课后教学的 Scratch 编程社区，融合「创作 + 社区 + 刷题 + 作业 + 积分 + AI」。

### 核心差异化

1. **sb3 深度解析** — 自动解压 .sb3 → 解析完整 JSON → 提取程序链、角色、背景、声音、变量、列表、广播等全部素材及属性 → 供 AI/教师分析
2. **AI 原生** — 架构层面为 AI 设计：AI 点评、AI 判题、AI 出题、AI 学情分析
3. **课后教学闭环** — 作业布置 → 学生提交 → 自动/手动批改 → 数据统计 → 学情报告
4. **班级宠物积分** — 趣味化的积分激励体系

> 当前阶段只做 Scratch，架构预留扩展接口，后续可加 Python、C++ 等。

| 角色 | 核心场景 |
|---|---|
| **学生** | 创作 Scratch 作品、发布分享、刷题练习、提交作业、查看积分/宠物/排行榜 |

### 模块 C：sb3 解析引擎 (SB3 Parser Service) ⭐ 核心

**这是本平台的核心差异化能力。**

#### 解析流程

```
输入: .sb3 文件 (本质是 ZIP 压缩包)

Step 1: 解压
  .sb3 → ZIP 解压 → project.json + 素材文件(png/wav/svg/mp3)

Step 2: 解析 project.json
  提取以下全部信息:

  ┌─ 项目元信息 ──────────────────────────┐
  │ • 项目名称、作者                       │
  │ • Scratch 版本                        │
  │ • 画布尺寸、帧率                       │
  └────────────────────────────────────────┘

  ┌─ 角色 (Sprites) ──────────────────────┐
  │ • 角色名称、坐标(x,y)、大小、方向      │
  │ • 可见性、旋转模式                      │
  │ • 当前造型                              │
  │ • 所有造型列表(含 md5/ext/尺寸)         │
  │ • 所有声音列表(含 md5/ext/时长)         │
  │ • 该角色上的所有积木(程序链)            │
  │ • 该角色的私有变量/列表                  │
  └────────────────────────────────────────┘

  ┌─ 舞台 (Stage) ────────────────────────┐
  │ • 所有背景造型列表                      │
  │ • 背景切换逻辑                          │
  │ • 全局变量/列表                         │
  │ • 全局积木(程序链)                      │
  │ • 扩展积木列表                          │
  └────────────────────────────────────────┘

  ┌─ 程序链 (Scripts) ────────────────────┐
  │ • 每条程序链的触发方式(绿旗/按键/点击等)│
  │ • 完整积木序列(含嵌套)                  │
  │ • 积木类型统计(循环/条件/事件/运动等)    │
  │ • 积木参数值                            │
  │ • 程序复杂度分析                        │
  └────────────────────────────────────────┘

  ┌─ 广播 (Broadcasts) ──────────────────┐
  │ • 广播消息列表                          │
  │ • 发送/接收关系                         │
  │ • 角色间通信链路                        │
  └────────────────────────────────────────┘

  ┌─ 素材文件 ─────────────────────────────┐
  │ • 所有 png/svg 文件(造型/背景)          │
  │ • 所有 wav/mp3 文件(声音)               │
  │ • 各素材的 md5、尺寸、时长              │
  └────────────────────────────────────────┘

Step 3: 结构化输出
  生成标准化 JSON → 存入 MongoDB
  供以下场景使用:
    ├── OJ 判题: 注入测试用例 → 运行 → 比对
    ├── AI 点评: 发送结构化数据给 AI 分析
    ├── 教师批改: 展示程序结构/素材使用/复杂度
    └── 数据统计: 分析学生编程习惯/知识掌握度
```

## 五、关键业务流程

### 流程 1：sb3 解析 + AI 点评

```
学生提交 .sb3 文件
  → sb3-parser 服务接收
  → ZIP 解压 → project.json + 素材文件
  → 解析 JSON → 提取完整结构
  → 存入 MongoDB
  → 发送结构化数据给 AI 服务
    → AI 分析: 程序结构/创意性/复杂度/改进建议
    → 返回点评文本
  → 展示给学生/教师:
    ├── 作品预览(舞台渲染)
    ├── 程序结构树(可视化)
    ├── 素材使用清单
    ├── 复杂度评分
    └── AI 点评 + 改进建议
```

### 流程 2：OJ 刷题

```
学生选择题目 → 在线 Scratch 编辑器
  → 完成编程 → 提交 .sb3
  → sb3-parser 解析
  → 判题引擎:
    ├── 算法题: 注入测试用例 → 运行 scratch-vm → 比对输出
    └── 创意题: AI 分析 sb3 结构 → 评分
  → 返回结果(AC/WA/TLE + 各测试点)
    ├── AC → +积分 → 更新排行榜 → 宠物喂养
    └── WA → +错题本 → AI 提示
```

### 流程 3：作业布置与批改

```
教师:
  选择课程单元 → 布置作业
  → 指定班级/截止时间/评分标准
  → 通知学生

学生:
  收到作业通知 → 在线创作 Scratch
  → 提交 .sb3 → 自动解析
  → 系统自动评分(如有关联题目)
  → 等待教师批改

教师批改:
  查看学生作品列表
  → 打开某作品 → 看到:
    ├── 作品预览(可运行)
    ├── sb3 解析结果(程序结构/素材/复杂度)
    ├── AI 预评(自动点评)
    └── 学生历史数据对比
  → 打分 + 写评语
  → 学生收到批改通知

数据统计:
  班级作业提交率/正确率/平均分
  → 生成学情报告
```
