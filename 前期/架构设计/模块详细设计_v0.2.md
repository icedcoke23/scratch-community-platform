> 日期：2026-04-23 | 基于 v0.1 架构 + v0.2 系统设计草案
> 变化：从 12 个业务模块精简为 **6 个模块 + 2 个共享库**

```

Layer 3 (教学模块):  judge(含 problem) / classroom
Layer 4 (系统模块):  system

@Service
public class ProjectService {
    // 创建项目
    ProjectVO createProject(Long userId, CreateProjectDTO dto)
    // dto: { title, description }
    // return: { id, title, status: "draft" }

    // 获取项目
    ProjectVO getProject(Long projectId)
    Page<ProjectVO> getMyProjects(Long userId, Pageable page)

    // 更新项目信息
    void updateProject(Long projectId, Long userId, UpdateProjectDTO dto)
    // dto: { title?, description?, coverUrl? }

    // 删除项目
    void deleteProject(Long projectId, Long userId)

    // 保存 sb3 → 自动触发解析
    ProjectVO saveProject(Long projectId, Long userId, MultipartFile sb3File) {
        // 1. 上传 sb3 到 MinIO
        String sb3Url = fileUploadUtils.upload(sb3File, "sb3");
        // 2. 调用 sb3-parser 解析
        SB3ParseResult parseResult = SB3Parser.parse(sb3File.getInputStream());
        // 3. 存解析结果到 MySQL (JSON 字段)
        project.setSb3Url(sb3Url);
        project.setParseResult(JSON.toJSONString(parseResult));
        project.setCoverUrl(extractCover(parseResult));
        project.setBlockCount(parseResult.getComplexity().getTotalBlocks());
        project.setComplexityScore(parseResult.getComplexity().getScore());
        projectMapper.updateById(project);
        return project.toVO();
    }

    // 获取解析结果
    SB3ParseResult getParseResult(Long projectId)

    // 发布
    void publishProject(Long projectId, Long userId) {
        // 1. 敏感词审核
        sensitiveWordFilter.check(project.getTitle() + project.getDescription());
        // 2. 更新状态为 published
        project.setStatus("published");
        projectMapper.updateById(project);
    }

    // 下载 sb3
    String getSb3DownloadUrl(Long projectId)
}
```

### 5.2 题目类型

| 类型 | type 值 | 判题方式 | 说明 |
|---|---|---|---|
| Scratch 算法题 | `scratch_algo` | 沙箱执行 + 测试用例比对 | 核心题型 |
| 选择题 | `choice` | 直接比对答案 | 快速检测知识 |
| 判断题 | `true_false` | 直接比对 | 选择题变体 |

### 5.3 服务层核心方法

```java
@Service

    // 题目 CRUD
    ProblemVO createProblem(Long userId, CreateProblemDTO dto)
    ProblemVO getProblem(Long problemId)
    Page<ProblemVO> listProblems(ProblemQueryDTO query, Pageable page)
    void updateProblem(Long problemId, Long userId, UpdateProblemDTO dto)
    void deleteProblem(Long problemId, Long userId)

    // 题目统计
    ProblemStatsVO getProblemStats(Long problemId)
}

@Service
public class JudgeService {
    // 提交
    SubmissionVO submit(Long userId, SubmitDTO dto) {
        // dto: { problemId, sb3File? (算法题), answer? (选择题) }
        // 1. 校验题目存在
        // 2. 创建 submission 记录 (status=pending)
        // 3. 异步判题
        // 4. 返回 submission 基本信息
    }

    // 异步判题
    @Async
    void judgeAsync(Submission submission) {
        // 1. 根据题目类型选择策略
        Problem problem = problemMapper.getById(submission.getProblemId());
        JudgeStrategy strategy = strategyFactory.getStrategy(problem.getType());

        // 2. 执行判题
        JudgeResult result = strategy.judge(submission, problem);

        // 3. 更新结果
        submission.setStatus(result.getStatus()); // AC/WA/TLE/RE
        submission.setScore(result.getScore());
        submission.setDetails(JSON.toJSONString(result.getDetails()));
        submission.setTimeUsed(result.getTimeUsed());
        submissionMapper.updateById(submission);
    }

    // 查询结果
    SubmissionVO getSubmission(Long submissionId)
    Page<SubmissionVO> getMySubmissions(Long userId, Pageable page)
    Page<SubmissionVO> getProblemSubmissions(Long userId, Long problemId, Pageable page)
}
```

### 5.4 判题流程

```
用户提交
  │
  ▼
创建 submission (status=pending)
  │
  ▼
异步判题 ─── 根据题目类型选择策略
  │
  ├── scratch_algo → ScratchAlgoStrategy
  │   ├── 1. 上传 sb3 到沙箱服务
  │   ├── 2. 注入测试用例
  │   ├── 3. 执行 scratch-vm
  │   ├── 4. 比对输出
  │   └── 5. 返回 AC/WA/TLE/RE + 分数
  │
  └── choice / true_false → ChoiceStrategy
      ├── 1. 读取用户答案
      ├── 2. 与正确答案比对
      └── 3. 返回 AC/WA + 分数
  │
  ▼
更新 submission 结果
  │
  ▼
后续处理 (MVP 简化: 仅更新状态，无积分/排行榜)
```

### 5.5 与其他模块的交互

```
judge → judge-core 库       (调用 JudgeCore 执行判题)
judge → sb3-parser 库       (解析 sb3 文件)
judge → common/SensitiveWordFilter (题目内容审核)
judge ← classroom 模块      (作业可关联题目，提交后触发判题)
```

- ❌ 错误分类/题解

    private String spriteName;                // 所属角色 ("_stage_" = 舞台)

    private List<String> usedBlockTypes;      // 使用的积木类型

    private int scriptCount;                  // 程序链数
    private int maxNestingDepth;              // 最大嵌套深度
    private int uniqueBlockTypes;             // 不同积木类型数
    private int variableCount;                // 变量数
    private int broadcastCount;               // 广播数
    private double score;                     // 综合评分 (0-100)
    private String level;                     // "入门" / "基础" / "进阶" / "高级"

### 8.4 解析流程

```
Step 1: SB3Unzipper.unzip(inputStream)
  → project.json (String) + assets/ (Map<String, byte[]>)

Step 2: ProjectJsonParser.parse(projectJson)
  → targets: List<Target>

Step 3: 并行提取
  ├── SpriteExtractor.extract(targets) → List<SpriteInfo>
  ├── VariableExtractor.extract(targets) → List<VariableInfo>
  ├── BroadcastAnalyzer.analyze(targets) → List<BroadcastRelation>
  └── BlockChainBuilder.build(targets) → List<ScriptChain>

Step 4: ComplexityCalculator.calculate(allData) → ComplexityScore

Step 5: 输出 → SB3ParseResult
```

### 8.5 复杂度评分算法

```
score = weighted_sum(
    totalBlocks * 1.0,
    scriptCount * 3.0,
    maxNestingDepth * 5.0,
    uniqueBlockTypes * 2.0,
    variableCount * 2.5,
    broadcastCount * 2.0,
    hasLoop * 5.0,
    hasCondition * 5.0,
    hasClone * 8.0,
    hasCustomBlock * 10.0
) / max_possible_score * 100

level = score < 20 ? "入门" : score < 45 ? "基础" : score < 70 ? "进阶" : "高级"
```

```java
public class ScratchAlgoStrategy implements JudgeStrategy {
    public boolean supports(String type) {
        return "scratch_algo".equals(type);
    }

    public JudgeResult judge(Submission submission, Problem problem) {
        // 1. 构建沙箱请求
        SandboxRequest req = SandboxRequest.builder()
            .projectUrl(submission.getSb3Url())
            .testCases(problem.getTestCases())
            .timeLimit(30000)   // 30s
            .memoryLimit(512)   // 512MB
            .build();

        // 2. 调用沙箱
        SandboxResponse resp = sandboxClient.submit(req);

        // 3. 解析结果
        return resultParser.parse(resp);
    }
}
```

### 9.4 ChoiceStrategy

```java
public class ChoiceStrategy implements JudgeStrategy {
    public boolean supports(String type) {
        return "choice".equals(type) || "true_false".equals(type);
    }

    public JudgeResult judge(Submission submission, Problem problem) {
        // 直接比对答案
        boolean correct = problem.getAnswer().equals(submission.getAnswer());
        JudgeResult result = new JudgeResult();
        result.setStatus(correct ? JudgeStatus.AC : JudgeStatus.WA);
        result.setScore(correct ? problem.getScore() : 0);
        return result;
    }
}
```

@Component
public class SensitiveWordFilter {
    // AC 自动机实现
    private Automaton automaton;

    @PostConstruct
    void init() {
        // 从数据库加载敏感词库
        List<String> words = sensitiveWordMapper.selectAll();
        automaton = new Automaton(words);
    }

    // 检查内容
    void check(String content) {
        List<String> hits = automaton.match(content);
        if (!hits.isEmpty()) {
            throw new BizException(60001, "内容包含违规信息: " + hits.get(0));
        }
    }

    // 过滤内容 (替换为 *)
    String filter(String content)

    // 刷新词库 (管理员更新后调用)
    void reload()
}
