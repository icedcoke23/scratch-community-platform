### 5.4 解析流程伪代码

```java
public SB3ParseResult parse(InputStream input) {
    // Step 1: 解压
    Map<String, byte[]> files = SB3Unzipper.unzip(input);
    String projectJson = new String(files.get("project.json"));

    // Step 2: 解析 JSON
    JSONObject root = JSON.parseObject(projectJson);
    JSONArray targets = root.getJSONArray("targets");

    // Step 3: 提取舞台和角色
    StageInfo stage = SpriteExtractor.extractStage(targets.getJSONObject(0));
    List<SpriteInfo> sprites = SpriteExtractor.extractSprites(targets);

    // Step 4: 构建程序链
    List<ScriptChain> scripts = BlockChainBuilder.buildAll(targets);

    // Step 5: 提取变量/列表/广播
    List<VariableInfo> vars = VariableExtractor.extract(targets);
    List<ListInfo> lists = VariableExtractor.extractLists(targets);
    List<BroadcastRelation> broadcasts = BroadcastAnalyzer.analyze(targets);

    // Step 6: 提取素材清单
    List<AssetInfo> assets = extractAssets(targets, files);

    // Step 7: 计算复杂度
    ComplexityScore complexity = ComplexityCalculator.calculate(
        scripts, vars, lists, broadcasts, assets
    );

    // Step 8: 组装结果
    return new SB3ParseResult(stage, sprites, scripts, vars, lists,
                              broadcasts, assets, complexity);
}
```

### 5.5 复杂度评分算法

```
score = weighted_sum(
    totalBlocks * 1.0,          // 积木数量 (基础分)
    scriptCount * 3.0,          // 程序链数 (多链 = 多逻辑)
    maxNestingDepth * 5.0,      // 嵌套深度 (深度 = 复杂逻辑)
    uniqueBlockTypes * 2.0,     // 积木种类 (多样性)
    variableCount * 2.5,        // 变量使用 (数据管理)
    listCount * 3.0,            // 列表使用 (数据结构)
    broadcastCount * 2.0,       // 广播使用 (角色通信)
    hasLoop * 5.0,              // 有循环
    hasCondition * 5.0,         // 有条件判断
    hasClone * 8.0,             // 有克隆 (高级概念)
    hasCustomBlock * 10.0       // 有自制积木 (函数概念)
) / max_possible_score * 100

level = score < 20 ? "入门" : score < 45 ? "基础" : score < 70 ? "进阶" : "高级"
```

```java
        String url = fileService.uploadSb3(sb3File);
        // 2. 调用 sb3-parser 解析
        SB3ParseResult parseResult = SB3Parser.parse(sb3File.getInputStream());

        mongoTemplate.save(parseResult);

        auditService.audit(project);
```

### 10.3 判题流程

```java
@Service
public class JudgeService {
    submit(Long userId, SubmitDTO dto) -> SubmissionVO {
        // 1. 创建 submission 记录 (status=pending)
        Submission sub = createSubmission(userId, dto);

        // 2. 异步判题 (MQ 或 @Async)
        judgeAsync(sub);

        return sub.toVO();
    }

    @Async
    void judgeAsync(Submission sub) {
        // 1. 获取题目
        Problem problem = problemMapper.getById(sub.getProblemId());

        // 2. sb3 解析 (如果是 Scratch 题)
        if (problem.isScratch()) {
            SB3ParseResult parse = SB3Parser.parse(sub.getSb3InputStream());
            sub.setParseResultId(parse.getId());
        }

        // 3. 选择判题策略
        JudgeStrategy strategy = strategyFactory.getStrategy(problem.getType());

        // 4. 执行判题
        JudgeResult result = strategy.judge(sub, problem);

        // 5. 更新结果
        sub.setResult(result.getStatus());
        sub.setScore(result.getScore());
        sub.setTestResults(result.getDetails());
        submissionMapper.updateById(sub);

        // 6. 触发后续
        if (result.isAC()) {
            pointsService.addPoints(sub.getUserId(), Action.AC_PROBLEM, problem.getScore());
            rankService.addScore(sub.getUserId(), "weekly_problems", 1);
        } else {
            wrongBookService.record(sub.getUserId(), sub.getProblemId());
        }
    }
}
```

### 10.4 Scratch 算法题判题 (ScratchAlgoStrategy)

```java
public class ScratchAlgoStrategy implements JudgeStrategy {
    public JudgeResult judge(Submission sub, Problem problem) {
        // 1. 调用判题沙箱
        SandboxRequest req = new SandboxRequest();
        req.setProjectUrl(sub.getSb3Url());
        req.setTestCases(problem.getTestCases());
        req.setTimeLimit(problem.getTimeLimit());
        req.setMemoryLimit(problem.getMemoryLimit());

        SandboxResponse resp = sandboxClient.submit(req);

        // 2. 解析结果
        return ResultParser.parse(resp);
    }
}
```

### 10.5 Scratch 创意题判题 (ScratchCreativeStrategy)

```java
public class ScratchCreativeStrategy implements JudgeStrategy {
    public JudgeResult judge(Submission sub, Problem problem) {
        // 1. 调用 AI Gateway
        AIJudgeResponse aiResp = aiGateway.creativeJudge(
            sub.getParseResultId(),
            problem.getRubric()
        );

        // 2. 组装结果
        JudgeResult result = new JudgeResult();
        result.setStatus(JudgeStatus.AI_SCORED);
        result.setScore(aiResp.getScore());
        result.setDetails(aiResp.getDimensions());
        return result;
    }
```
