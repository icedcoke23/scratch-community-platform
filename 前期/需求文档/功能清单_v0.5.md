# 📝 功能清单 v0.5

> 日期: 2026-04-23 | 基于 v0.4 深度精简，聚焦真正可交付的 MVP

---

## 0. 设计原则

| 原则 | 说明 |
|------|------|
| **真正的 MVP** | 6-8 周、2-3 人可完成的最小可行产品 |
| **核心循环** | 学生注册 → 创作 → 提交 → 判题 → 看结果 → 社区分享 → 获得反馈 |
| **6 模块** | 从 v0.4 的 14 模块砍到 6 个核心模块 |
| **只做 P0** | 每个模块只保留最高优先级功能 |
| **延后 ≠ 砍掉** | 所有延后功能保留在对应 Phase 中，不做不代表永远不做 |

### v0.4 → v0.5 精简对照

| v0.4 模块 | v0.5 处理 | 说明 |
|-----------|----------|------|
| 用户系统 | ✅ 保留 | P0: 注册/登录/角色/个人资料 |
| Scratch 创作引擎 | ✅ 保留 | 砍掉 Remix 归属链/版本历史/协作编辑/模板，只做编辑器+保存+导出+预览 |
| 社区系统 | ✅ 保留 | 砍掉 Wilson Score/Feed/发现页/创作挑战，只做发布/浏览(最新+最热)/点赞/评论 |
| 判题系统 | ✅ 保留 | 只做 Level 1 headless + 选择题，砍掉 AI 判题/三级降级 |
| 教师管理 | ✅ 保留(P1) | 只保留基础班级管理，延后 Rubric/学情报告/智能出题 |
| 系统管理 | ✅ 保留(P2) | 内容审核(敏感词)/用户管理/站内通知 |
| 推荐引擎 | ❌ 合并 | 合并到社区模块，只做简单排序（最新/最热） |
| 竞赛系统 | ❌ 延后 | 完全延后到 Phase 3 |
| 竞赛题库 | ❌ 延后 | 完全延后到 Phase 2 |
| 用户成长路径 | ⚠️ 精简 | 只保留基础积分和等级，延后徽章/认证/证书 |
| 学习路径 | ❌ 延后 | 延后到 Phase 2 |

| 素材库 | ⚠️ 精简 | 只做官方素材，延后用户上传 |
| 通知系统 | ⚠️ 精简 | 只做站内通知，延后邮件/推送 |

## 3. MVP 路线图

### Phase 0.5: 技术验证 Spike（1-2 周）

| 任务 | 时间 | 说明 |
|------|------|------|
| scratch-vm headless 验证 | 2-3 天 | Node.js 18 + jsdom + node-canvas，确认可行性 |
| 选择题判题方案验证 | 1-2 天 | 自动判分逻辑 + 结果存储 |

## 4. 功能依赖关系（v0.5 精简版）

```
用户系统 ─┬─→ 创作引擎 ─→ 社区系统
           │
           ├─→ 判题系统
           │
           ├─→ 教师管理
           │
           └─→ 通知系统

创作引擎 ─→ 判题系统（项目提交后触发判题）
判题系统 ─→ 教师管理（作业需要判题结果）
用户系统 ─→ 社区系统（发布/点赞/评论需要用户身份）
用户系统 ─→ 通知系统（通知需要用户身份）
```

**MVP 关键路径:** 用户系统 → 创作引擎 → 判题系统 → 社区系统

**最小闭环:** 注册 → 登录 → 打开编辑器 → 创作项目 → 保存 → 发布到社区 → 被点赞/评论 → 收到通知

## 5. 数据模型（v0.5 MVP 最小集）

```sql
-- 用户表
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE,
    email VARCHAR(100) UNIQUE,
    phone VARCHAR(20) UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    avatar_url VARCHAR(500),
    nickname VARCHAR(50),
    bio TEXT,
    role ENUM('student', 'teacher', 'admin') DEFAULT 'student',
    points INT DEFAULT 0 COMMENT '基础积分',
    level INT DEFAULT 1 COMMENT '基础等级',
    status ENUM('active', 'banned') DEFAULT 'active',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 项目表
CREATE TABLE projects (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    title VARCHAR(200) NOT NULL,
    description TEXT,
    cover_url VARCHAR(500),
    sb3_url VARCHAR(500) NOT NULL COMMENT 'sb3 文件存储路径',
    tags VARCHAR(500) COMMENT '标签，逗号分隔',
    like_count INT DEFAULT 0,
    comment_count INT DEFAULT 0,
    status ENUM('draft', 'published', 'hidden') DEFAULT 'draft',
    published_at DATETIME,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id),
    INDEX idx_published_at (published_at),
    INDEX idx_like_count (like_count DESC)
);

-- 点赞表
CREATE TABLE likes (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    project_id BIGINT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_user_project (user_id, project_id)
);

-- 评论表
CREATE TABLE comments (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    project_id BIGINT NOT NULL,
    content TEXT NOT NULL,
    status ENUM('visible', 'hidden') DEFAULT 'visible',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_project_id (project_id)
);

-- 题目表
CREATE TABLE problems (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(200) NOT NULL,
    description TEXT NOT NULL,
    type ENUM('scratch', 'choice') NOT NULL COMMENT 'scratch=Scratch编程题, choice=选择题',
    difficulty ENUM('easy', 'medium', 'hard') DEFAULT 'easy',
    sb3_template_url VARCHAR(500) COMMENT 'Scratch题目的模板项目',
    choice_options JSON COMMENT '选择题选项 [{"key":"A","text":"..."},{"key":"B","text":"..."}]',
    correct_answer VARCHAR(100) COMMENT '选择题正确答案',
    test_cases JSON COMMENT 'Scratch题的测试用例',
    status ENUM('active', 'inactive') DEFAULT 'active',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    created_by BIGINT
);

-- 提交/判题记录表
CREATE TABLE submissions (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    problem_id BIGINT NOT NULL,
    project_id BIGINT COMMENT 'Scratch题关联的项目ID',
    answer VARCHAR(100) COMMENT '选择题提交的答案',
    result ENUM('pending', 'pass', 'fail', 'error') DEFAULT 'pending',
    score INT DEFAULT 0,
    details JSON COMMENT '判题详情（通过/失败的用例等）',
    submitted_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    judged_at DATETIME,
    INDEX idx_user_problem (user_id, problem_id)
);

-- 班级表
CREATE TABLE classes (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    teacher_id BIGINT NOT NULL,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    invite_code VARCHAR(20) NOT NULL UNIQUE,
    student_count INT DEFAULT 0,
    status ENUM('active', 'archived') DEFAULT 'active',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 班级学生关系表
CREATE TABLE class_students (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    class_id BIGINT NOT NULL,
    student_id BIGINT NOT NULL,
    joined_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_class_student (class_id, student_id)
);

-- 作业表
CREATE TABLE assignments (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    class_id BIGINT NOT NULL,
    teacher_id BIGINT NOT NULL,
    title VARCHAR(200) NOT NULL,
    description TEXT,
    problem_ids JSON COMMENT '关联的题目ID列表',
    due_date DATETIME,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 作业提交表
CREATE TABLE assignment_submissions (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    assignment_id BIGINT NOT NULL,
    student_id BIGINT NOT NULL,
    submission_id BIGINT COMMENT '关联的判题提交ID',
    score INT,
    comment TEXT COMMENT '教师评语',
    status ENUM('pending', 'graded') DEFAULT 'pending',
    submitted_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    graded_at DATETIME,
    UNIQUE KEY uk_assignment_student (assignment_id, student_id)
);

-- 通知表
CREATE TABLE notifications (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    type ENUM('system', 'like', 'comment', 'assignment', 'grade') NOT NULL,
    title VARCHAR(200),
    content TEXT,
    related_id BIGINT COMMENT '关联的对象ID',
    is_read TINYINT DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_user_read (user_id, is_read)
);

-- 敏感词表
CREATE TABLE sensitive_words (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    word VARCHAR(100) NOT NULL UNIQUE,
    level ENUM('low', 'medium', 'high') DEFAULT 'medium',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

| Remix 归属链 | P1 | P2 | 增加开发复杂度，非核心 |
| 三级降级判题 | P1 | P2+ | Level 1 先跑通 |
| AI 判题 | P4 | P4 | 不影响核心循环 |
| 收藏分离 | P1 | P2 | 点赞够用 |
| 关注 Feed | P3 | P3 | 需要用户规模 |
| 数据看板 | P2 | P2 | 非核心教学循环 |

### 保留的核心循环

```
学生注册 ─→ 打开编辑器 ─→ 创作 Scratch 项目
    │                          │
    │                          ↓
    │                     保存/导出
    │                          │
    │                          ↓
    │               提交到判题系统（选择题/Scratch题）
    │                          │
    │                          ↓
    │                     查看判题结果
    │                          │
    │                          ↓
    └──────── 发布到社区 ←─────┘
                    │
                    ↓
              被浏览/点赞/评论
                    │
                    ↓
              收到站内通知 ─→ 继续创作
```
