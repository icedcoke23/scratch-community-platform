# 移动端 App 开发方案

> 版本: v1.0 | 日期: 2026-04-28
> 状态: 技术选型阶段

---

## 一、需求分析

### 目标用户

| 用户群 | 核心需求 | 优先级 |
|--------|----------|--------|
| 学生 | 浏览作品、点赞评论、查看作业 | P0 |
| 教师 | 发布作业、查看学情、批改 | P1 |
| 家长 | 查看孩子学习进度 | P2 |

### 移动端特有功能

- 推送通知（作业提醒、点赞通知）
- 离线缓存（已浏览的作品可离线查看）
- 分享到微信/QQ
- 拍照上传（作业提交）
- 摇一摇（随机推荐作品）

## 二、技术选型

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **uni-app (Vue)** | 代码复用率高，团队已有 Vue 经验 | 性能中等，原生能力受限 | ⭐⭐⭐⭐⭐ |
| **Flutter** | 性能优秀，UI 一致性好 | 需要学习 Dart，生态较小 | ⭐⭐⭐ |
| **React Native** | 社区成熟，热更新 | 性能不如原生，调试困难 | ⭐⭐⭐ |
| **原生开发** | 性能最优，功能完整 | 开发成本高，需要双端维护 | ⭐⭐ |

**推荐方案: uni-app**

理由:
1. 前端团队已有 Vue 3 + TypeScript 经验，学习成本最低
2. 可复用现有 Vue 组件和 API 调用逻辑
3. 一套代码支持 iOS + Android + H5 + 小程序
4. DCloud 生态成熟，插件丰富

## 五、推送通知方案

### 使用 uni-push 2.0

```javascript
// 获取推送 CID
uni.getPushClientId({
  success: (res) => {
    const cid = res.cid
    // 上报 CID 到后端
    api.reportPushToken({ platform: 'app', token: cid })
  }
})

// 监听推送消息
uni.onPushMessage((res) => {
  const { type, payload } = res.data
  // 处理不同类型的推送
  switch (type) {
    case 'homework':
      // 跳转到作业详情
      break
    case 'like':
      // 跳转到作品详情
      break
    case 'comment':
      // 跳转到评论
      break
  }
})
```

## 七、与 Web 端共享代码

### 可直接复用

- API 调用层 (`api/*.ts`)
- 状态管理逻辑 (`stores/*.ts`)
- 工具函数 (`utils/*.ts`)
- 类型定义 (`types/*.ts`)

### 需要适配

- UI 组件（Element Plus → uni-ui）
- 路由配置（Vue Router → pages.json）
- 样式（CSS → rpx 单位）
- 浏览器 API（部分需要条件编译）

## 八、发布流程

```
开发 → 测试 → 灰度发布 → 全量发布

1. 开发: 本地 HBuilderX 开发调试
2. 测试: 真机调试 + 云测试
3. 灰度: 先发布到 10% 用户
4. 全量: 无问题后全量发布
5. 更新: 使用 wgt 热更新
```
