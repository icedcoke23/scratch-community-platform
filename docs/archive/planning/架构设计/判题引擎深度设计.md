# 🔍 判题引擎深度设计

> 日期：2026-04-23
> 核心洞察：Scratch 程序大量是效果导向，非输出导向，需要双轨判题

---

## 一、问题分析

### 传统 OJ 判题的局限

```
传统 OJ (LeetCode/Codeforces):
  输入 → 程序 → 输出 → 比对标准输出 → AC/WA
  ✅ 适合: 算法题、数学题、数据处理题

Scratch 的现实:
  ├── 算法题: "输入 3，输出 6（3的阶乘）" → 可以用传统判题
  ├── 效果题: "画一个会动的太阳" → 没有标准输出，怎么办？
  ├── 交互题: "做一个接球游戏" → 效果取决于用户操作
  ├── 创意题: "用 Scratch 讲一个故事" → 完全主观
  └── 动画题: "让小猫跳舞" → 需要看视觉效果
```

### Scratch 题目类型分布（实际场景）

| 类型 | 占比 | 判题方式 |
|---|---|---|
| 算法题（有输入输出） | ~20% | 传统测试用例 |
| 效果题（画图/动画/游戏） | ~45% | **AI 判题** |
| 交互题（需要用户操作） | ~20% | **AI 判题 + 行为分析** |
| 创意题（开放式） | ~15% | **AI 判题 + 教师复审** |

**结论：超过 80% 的 Scratch 题目需要 AI 参与判题。**

## 三、轨道 A：传统判题（测试用例）

> 适用于：算法题（有明确输入输出）

### 判题流程

```
1. sb3 解析 → 提取积木结构
2. 向 scratch-vm 注入输入:
   ├── 方式 1: 修改"回答"积木的值 (替换"询问并等待"的回答)
   ├── 方式 2: 修改变量初始值 (替换全局变量的默认值)
   └── 方式 3: 模拟按键/点击事件
3. 运行 scratch-vm (headless)
4. 等待执行完成或超时
5. 捕获输出:
   ├── "说…()" 积木的输出文本
   ├── 全局变量的最终值
   ├── 列表的最终内容
   └── 精灵状态 (坐标/可见性/造型)
6. 比对标准输出
7. 返回 AC/WA/TLE
```

## 四、轨道 B：AI 判题（效果分析）⭐ 核心创新

> 适用于：效果题、交互题、创意题、动画题、游戏题

### 4.1 判题流程

```
Step 1: sb3 深度解析
  .sb3 → SB3Parser.parse()
  → 输出:
    ├── 项目元信息 (名称/画布/帧率)
    ├── 角色列表 (名称/坐标/造型/声音/变量)
    ├── 舞台信息 (背景/全局变量)
    ├── 完整程序链 (每条链的积木序列)
    ├── 积木类型统计
    ├── 广播关系图
    └── 复杂度评分

Step 2: 构造 AI 判题请求
  ├── 程序链的自然语言描述 (将积木翻译为可读描述)
  ├── 题目要求 (预期效果描述)
  ├── 判题标准 (评分维度+权重)
  └── 可选: 预期效果截图/描述

Step 3: 发送给 AI Gateway
  → POST /ai/judge
  → AI 分析程序逻辑 + 对比预期效果
  → 返回: 评分 + 各维度分数 + 详细点评 + 改进建议

Step 4: 结果处理
  ├── 高置信度 (>0.8): 直接采信
  ├── 中置信度 (0.5-0.8): 标记为"需教师确认"
  └── 低置信度 (<0.5): 转人工审核
```

### 4.2 程序链 → 自然语言转换

**这是 AI 判题的关键桥梁。** 将 Scratch 积木结构转换为 AI 能理解的自然语言描述。

```java
public class BlockTranslator {

    /**
     * 将一条程序链翻译为自然语言描述
     */
    public String translate(ScriptChain chain) {
        StringBuilder sb = new StringBuilder();

        // 触发方式
        sb.append("【触发】").append(describeTrigger(chain)).append("\n");
        sb.append("【角色】").append(chain.getSpriteName()).append("\n");
        sb.append("【程序】\n");

        // 递归翻译积木链
        translateBlock(chain.getRoot(), sb, 0);

        return sb.toString();
    }

    private void translateBlock(BlockNode node, StringBuilder sb, int indent) {
        if (node == null) return;

        String prefix = "  ".repeat(indent);
        sb.append(prefix).append(describeBlock(node)).append("\n");

        // 翻译子分支 (循环体/条件分支)
        for (BlockNode branch : node.getBranches()) {
            translateBlock(branch, sb, indent + 1);
        }

        // 翻译同级积木
        translateBlock(node.getNext(), sb, indent);
    }

    private String describeBlock(BlockNode node) {
        String opcode = node.getOpcode();
        Map<String, Object> inputs = node.getInputs();
        Map<String, Object> fields = node.getFields();

        return switch (opcode) {
            case "motion_movesteps" -> "移动 " + inputs.get("STEPS") + " 步";
            case "motion_turnright" -> "右转 " + inputs.get("DEGREES") + " 度";
            case "motion_turnleft" -> "左转 " + inputs.get("DEGREES") + " 度";
            case "motion_gotoxy" -> "移到 x:" + inputs.get("X") + " y:" + inputs.get("Y");
            case "motion_glidesecstoxy" ->
                "滑行 " + inputs.get("SECS") + " 秒到 x:" + inputs.get("X") + " y:" + inputs.get("Y");
            case "looks_sayforsecs" ->
                "说 \"" + inputs.get("MESSAGE") + "\" " + inputs.get("SECS") + " 秒";
            case "looks_switchcostumeto" -> "切换造型为 " + inputs.get("COSTUME");
            case "looks_show" -> "显示";
            case "looks_hide" -> "隐藏";
            case "sound_play" -> "播放声音 " + inputs.get("SOUND_MENU");
            case "control_repeat" -> "重复 " + inputs.get("TIMES") + " 次:";
            case "control_repeat_until" -> "重复直到 " + describeCondition(inputs.get("CONDITION")) + ":";
            case "control_forever" -> "永远重复:";
            case "control_if" -> "如果 " + describeCondition(inputs.get("CONDITION")) + " 则:";
            case "control_if_else" -> "如果 " + describeCondition(inputs.get("CONDITION")) + " 则: 否则:";
            case "control_wait" -> "等待 " + inputs.get("DURATION") + " 秒";
            case "control_create_clone_of" -> "克隆 " + inputs.get("CLONE_OPTION");
            case "event_whenflagclicked" -> "当绿旗被点击";
            case "event_whenkeypressed" -> "当按下 " + inputs.get("KEY_OPTION") + " 键";
            case "event_whenthisspriteclicked" -> "当角色被点击";
            case "event_whenbroadcastreceived" -> "当接收到 " + inputs.get("BROADCAST_INPUT");
            case "event_broadcast" -> "广播 " + inputs.get("BROADCAST_INPUT");
            case "data_setvariableto" -> "设置 " + fields.get("VARIABLE") + " 为 " + inputs.get("VALUE");
            case "data_changevariableby" -> "将 " + fields.get("VARIABLE") + " 增加 " + inputs.get("VALUE");
            case "data_addtolist" -> "将 " + inputs.get("ITEM") + " 添加到列表 " + fields.get("LIST");
            case "operator_add" -> "(" + inputs.get("NUM1") + " + " + inputs.get("NUM2") + ")";
            case "operator_random" -> "随机数(" + inputs.get("FROM") + " 到 " + inputs.get("TO") + ")";
            case "operator_lt" -> "(" + inputs.get("OPERAND1") + " < " + inputs.get("OPERAND2") + ")";
            case "sensing_touchingobject" -> "碰到 " + inputs.get("TOUCHINGOBJECTMENU");
            case "pen_clear" -> "清空画笔";
            case "pen_penDown" -> "落笔";
            case "pen_setPenColorToColor" -> "设置画笔颜色为 " + inputs.get("COLOR");
            default -> "[" + opcode + "]";
        };
    }
}
```

### 4.5 AI 判题服务层

```java
@Service
public class AIJudgeService {

    @Autowired
    private AIGatewayClient aiGateway;

    @Autowired
    private BlockTranslator translator;

    /**
     * AI 判题主流程
     */
    public AIJudgeResult judge(Submission submission, Problem problem) {
        // 1. 获取 sb3 解析结果
        SB3ParseResult parse = sb3Parser.fromMongo(submission.getParseResultId());

        // 2. 将程序链翻译为自然语言
        String scriptsDetail = translateAllScripts(parse);

        // 3. 构造判题请求
        AIJudgeRequest request = new AIJudgeRequest();
        request.setProblemDescription(problem.getDescription());
        request.setRubric(problem.getRubric());           // 判题标准
        request.setProjectName(parse.getMeta().getName());
        request.setSprites(describeSprites(parse));
        request.setScriptCount(parse.getScripts().size());
        request.setTotalBlocks(parse.getComplexity().getTotalBlocks());
        request.setScriptsDetail(scriptsDetail);
        request.setBlockTypes(parse.getBlockStats().toString());
        request.setVariablesAndLists(describeVarsAndLists(parse));
        request.setBroadcastRelations(describeBroadcasts(parse));

        // 4. 调用 AI Gateway
        AIJudgeResponse response = aiGateway.judge(request);

        // 5. 置信度处理
        if (response.getConfidence() >= 0.8) {
            return AIJudgeResult.accepted(response);
        } else if (response.getConfidence() >= 0.5) {
            return AIJudgeResult.needsReview(response);
        } else {
            return AIJudgeResult.uncertain(response);
        }
    }

    /**
     * 将所有程序链翻译为自然语言
     */
    private String translateAllScripts(SB3ParseResult parse) {
        StringBuilder sb = new StringBuilder();

        // 舞台的程序链
        for (ScriptChain chain : parse.getStage().getScripts()) {
            sb.append("=== 舞台 ===\n");
            sb.append(translator.translate(chain));
            sb.append("\n");
        }

        // 各角色的程序链
        for (SpriteInfo sprite : parse.getSprites()) {
            for (ScriptChain chain : sprite.getScripts()) {
                sb.append("=== 角色: ").append(sprite.getName()).append(" ===\n");
                sb.append(translator.translate(chain));
                sb.append("\n");
            }
        }

        return sb.toString();
    }
}
```

### ScratchEffectStrategy（效果题判题）

```java
public class ScratchEffectStrategy implements JudgeStrategy {

    public JudgeResult judge(Submission sub, Problem problem) {
        // 1. AI 判题
        AIJudgeResult aiResult = aiJudgeService.judge(sub, problem);

        // 2. 根据置信度处理
        JudgeResult result = new JudgeResult();

        switch (aiResult.getStatus()) {
            case ACCEPTED:
                // 高置信度，直接采信
                result.setStatus(JudgeStatus.SCORED);
                result.setScore(aiResult.getScore());
                result.setDetails(aiResult.getDimensions());
                result.setAiComment(aiResult.getAnalysis());
                break;

            case NEEDS_REVIEW:
                // 中置信度，标记待教师确认
                result.setStatus(JudgeStatus.NEEDS_REVIEW);
                result.setScore(aiResult.getScore());      // 暂定分数
                result.setAiComment(aiResult.getAnalysis());
                result.setNeedsTeacherReview(true);
                break;

            case UNCERTAIN:
                // 低置信度，转人工
                result.setStatus(JudgeStatus.PENDING_REVIEW);
                result.setAiComment("AI 无法确定评判，请教师人工评分");
                break;
        }

        return result;
    }
}
```

```
判题:
  ├── 轨道 A: 注入输入 "3" "5" → 检查输出 "8" (40分)
  └── 轨道 B: AI 分析界面设计和代码质量 (60分)
```

### 混合判题策略

```java
public class ScratchHybridStrategy implements JudgeStrategy {

    public JudgeResult judge(Submission sub, Problem problem) {
        JudgeResult result = new JudgeResult();

        // 轨道 A: 测试用例判题 (如果有)
        if (problem.hasTestCases()) {
            JudgeResult algoResult = algoStrategy.judge(sub, problem);
            result.setAlgoScore(algoResult.getScore() * 0.4);  // 占 40%
            result.setAlgoDetails(algoResult.getDetails());
        }

        // 轨道 B: AI 判题
        AIJudgeResult aiResult = aiJudgeService.judge(sub, problem);
        result.setAiScore(aiResult.getScore() * 0.6);          // 占 60%
        result.setAiDetails(aiResult.getDimensions());

        // 合并
        result.setTotalScore(result.getAlgoScore() + result.getAiScore());
        result.setStatus(JudgeStatus.SCORED);

        return result;
    }
}
```

## 八、AI 判题质量保障

### 8.1 多轮验证

```
对于高分值题目 (score > 50):
  ├── 第 1 轮: AI 判题 (模型 A)
  ├── 第 2 轮: AI 判题 (模型 B，或同模型不同 Prompt)
  └── 两轮差异 > 15 分 → 转人工

对于普通题目:
  └── 单轮 AI 判题，置信度过低转人工
```

### 8.2 教师校准

```
教师批改时:
  ├── 教师评分 vs AI 评分 → 记录差异
  ├── 差异 > 15 分 → AI 评分被标记为"不准"
  ├── 累计偏差 → 触发 Prompt 调整
  └── 教师评分作为 ground truth → 持续优化 AI
```
