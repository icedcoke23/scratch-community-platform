## 3. MVP 路线图

### Phase 0.5: 技术验证（1-2 周）

- [ ] scratch-vm headless 技术验证 spike（2-3天）
  - 验证 Node.js 18 + jsdom + node-canvas 方案可行性
  - 测试 Polyfill（AudioContext mock, Canvas mock）
  - 确认不支持的功能范围（画笔/音乐/视频扩展）
- [ ] 确认判题沙箱的降级策略可行性
  - Level 1 → Level 2 → Level 3 降级流程验证
  - 各级别性能基准测试
- [ ] 搭建项目骨架
  - Spring Boot 3 + Java 17 后端骨架
  - Vue 3 + Vite + TypeScript 前端骨架
  - docker-compose 基础环境（MySQL + Redis + MinIO）

### Phase 1: 核心 MVP（6-8 周）

- [ ] 用户系统（注册/登录/角色）
- [ ] Scratch 在线编辑器（嵌入 scratch-gui）
- [ ] 项目保存/加载/导出
- [ ] Remix 基础功能（二次创作 + 原创者追溯）
- [ ] 基础社区（发布/浏览/点赞）
- [ ] Level 1 判题（scratch-vm headless）
- [ ] 素材库（官方素材）
- [ ] 通知系统（站内通知）
- [ ] 系统管理（内容审核/用户管理）

### Phase 2: 教学功能（4-6 周）

- [ ] 学习路径（课程体系/关卡/进度）
- [ ] 教师管理（班级/作业/批改）
- [ ] Level 2 静态分析判题
- [ ] 新作曝光池 + Wilson Score 热门排名
- [ ] 教师推荐置顶
- [ ] 数据看板（学习数据/趋势图）
- [ ] 判题历史

### Phase 3: 竞赛 & 扩展（4-6 周）

- [ ] 竞赛系统基础版（限时赛/排名赛/实时排名）
- [ ] 竞赛积分系统
- [ ] Remix 树展示
- [ ] 个性化推荐（基础版）
- [ ] Level 3 浏览器沙箱判题
- [ ] 按需引入 MongoDB / Elasticsearch

### Phase 4: 成长 & 深化（4-6 周）

- [ ] 用户成长路径可视化
- [ ] 里程碑 & 成就系统
- [ ] 等级系统 & 徽章墙
- [ ] 创意挑战赛
- [ ] 协作编辑（探索性）
- [ ] 数据导出 & 高级分析

## 4. 与 v0.2 变更摘要

| 变更项 | v0.2 | v0.3 |
|--------|------|------|
| 功能模块数 | 10 个 | **12 个**（+竞赛系统 +用户成长路径） |
| Remix | 无 | **新增**：二次创作/Remix 树/原创者追溯 |
| 社区推荐 | 基础排序 | **新增**：新作曝光池/Wilson Score/个性化推荐/教师置顶 |
| 判题系统 | 未分层 | **三级降级**：headless → 静态分析 → 浏览器沙箱 |
| 竞赛系统 | 无 | **新增**：限时赛/排名赛/创意挑战/实时排名/积分 |
| 成长路径 | 无 | **新增**：成长地图/里程碑/等级/徽章墙 |
| Phase 0.5 | 无 | **新增**：技术验证 spike（判题沙箱可行性 + 项目骨架） |
| MVP 路线图 | 3 阶段 | **5 阶段**（Phase 0.5 → Phase 4） |
| Phase 1 | 核心功能 | **+Remix 基础功能** |
| Phase 2 | 教学功能 | **+静态分析判题** |
| Phase 3 | 扩展 | **+竞赛系统基础版** |
| Phase 4 | 无 | **新增**：用户成长路径可视化 |

## 5. 功能依赖关系

```
用户系统 ─┬─→ 创作引擎 ─→ Remix ─→ 社区系统
           │
           ├─→ 判题系统 ─→ 学习路径
           │
           ├─→ 教师管理 ─→ 数据看板
           │
           ├─→ 竞赛系统 ─→ 用户成长路径
           │
           └─→ 通知系统
```

**关键路径：** 用户系统 → 创作引擎 → 判题系统 → 学习路径 → 竞赛系统 → 成长路径
