## 3. MVP 路线图

### Phase 0.5: 技术验证（1-2 周）

- [ ] scratch-vm headless 技术验证 spike（2-3天）
  - 验证 Node.js 18 + jsdom + node-canvas 方案可行性
  - 测试 Polyfill（AudioContext mock, Canvas mock）
  - 确认不支持的功能范围（画笔/音乐/视频扩展）
- [ ] 确认判题沙箱的降级策略可行性
  - Level 1 → Level 2 → Level 3 降级流程验证
  - 各级别性能基准测试
- [ ] 搭建项目骨架
  - Spring Boot 3 + Java 17 后端骨架
  - Vue 3 + Vite + TypeScript 前端骨架
  - docker-compose 基础环境（MySQL + Redis + MinIO）

### Phase 1: 核心 MVP（6-8 周）

**用户系统：**
- [ ] 注册/登录/角色管理
- [ ] 个人资料

**创作引擎：**
- [ ] Scratch 在线编辑器（嵌入 scratch-gui）
- [ ] 项目保存/加载/导出
- [ ] Remix 二次创作 + **归属链穿透（root_id）** — *借鉴 Scratch MIT*

**社区系统：**
- [ ] 项目发布/浏览
- [ ] **点赞 + 收藏分离** — *借鉴 GitHub/Pixiv*
- [ ] 评论/分享
- [ ] **新作曝光池（前 24h 保底曝光）** — *借鉴 Bilibili*
- [ ] **Wilson Score 热门排名** — *借鉴 Reddit/Bilibili*

**判题系统：**
- [ ] Level 1 判题（scratch-vm headless）

### Rating 计算（竞赛）

```
基于 Elo 系统:
  新 Rating = 旧 Rating + K × (实际排名 - 预期排名)

  K 因子:
    新用户 (< 20 场): K = 40
    普通用户: K = 20
    高分用户 (> 2100): K = 10

颜色标识:
  新手 (Newbie):        0-800    灰色
  入门 (Learner):       801-1200  绿色
  熟练 (Practitioner):  1201-1600 蓝色
  精通 (Proficient):    1601-2000 紫色
  大师 (Master):        2001+     金色
```

## 8. Wilson Score 算法说明

用于热门排名，避免小样本偏差：

```python
# Wilson Score 置信区间下界
def wilson_score(positive, total, confidence=0.95):
    """
    positive: 点赞数
    total: 总互动数（点赞+踩/浏览）
    confidence: 置信度，默认 95%
    """
    if total == 0:
        return 0
    z = 1.96  # 95% 置信度
    phat = positive / total
    return (phat + z*z/(2*total) - z * sqrt((phat*(1-phat)+z*z/(4*total))/total)) / (1 + z*z/total)

# 排序: 按 wilson_score 降序
# 效果: 100赞/100浏览 > 1000赞/10000浏览（前者质量更高）
```

## 9. 新作曝光池机制

```
新作品发布后:
├── 前 6 小时: 进入"新作曝光池"，获得保底曝光（推荐位轮播）
├── 6-24 小时: 根据初始互动数据（点赞率/收藏率/完播率）决定是否继续曝光
├── 24 小时后: 退出曝光池，进入正常排名算法
└── 特殊情况: 如果 24h 内互动数据优秀，可进入"热门候选池"

曝光池容量: 每次展示 20 个作品，每 30 分钟轮换一次
选择算法: 时间权重 + 随机因子（保证公平性）
```
