```javascript
const vm = new VirtualMachine();
await vm.loadProject(projectData);

// 注入测试用例
vm.runtime.on('QUESTION', (q) => {
  vm.runtime.emit('ANSWER', testCases[i].input);

const output = {
  variables: vm.runtime.targets[0].variables,

```

```
  ├── Service Worker 缓存 (离线可用)
  ├── 版本管理 (与后端 sb3-parser 版本一致)
  └── 预加载: 用户可能要创作时提前加载
```

## 七、sb3 解析方案

### 核心原理

```
.sb3 = ZIP 压缩包
  ├── project.json (核心数据)
  ├── abc123.png (造型)
  ├── def456.wav (声音)
  └── ...

解析流程:
  1. ZIP 解压 → project.json + 素材文件
  2. JSON.parse(project.json)
  3. 遍历 targets[] → 提取舞台 + 角色
  4. 遍历 blocks[] → 构建程序链 (从帽子积木开始)
  5. 统计分析 → 积木类型分布 / 嵌套深度 / 复杂度
  6. 输出 SB3ParseResult → 存 MongoDB
```

### 实现技术

| 步骤 | Java 实现 | 说明 |
|---|---|---|
| ZIP 解压 | java.util.zip | JDK 内置 |
| JSON 解析 | Jackson / FastJSON | 标准 JSON 库 |
| 积木链构建 | 自研 | 遍历 blocks Map，以 opcode 构建链 |
| 复杂度计算 | 自研 | 加权评分算法 |

## 九、部署方案

### 开发环境 (docker-compose.yml)

```yaml
services:
  mysql:
    image: mysql:8.0
    ports: ["3306:3306"]
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: scratch_community

  mongodb:
    image: mongo:7
    ports: ["27017:27017"]

  redis:
    image: redis:7-alpine
    ports: ["6379:6379"]

  minio:
    image: minio/minio
    ports: ["9000:9000", "9001:9001"]
    command: server /data --console-address ":9001"

  elasticsearch:
    image: elasticsearch:8.11.0
    ports: ["9200:9200"]
    environment:
      discovery.type: single-node
      xpack.security.enabled: "false"

  api:
    build: ./api
    ports: ["8080:8080"]
    depends_on: [mysql, mongodb, redis, minio, elasticsearch]

  judge-sandbox:
    build: ./judge-sandbox
    ports: ["8081:8081"]
    depends_on: [redis]
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock

  ai-gateway:
    build: ./ai-gateway
    ports: ["8082:8082"]
    depends_on: [redis]

  web:
    image: nginx:alpine
    ports: ["80:80"]
    volumes:
      - ./web/dist:/usr/share/nginx/html
      - ./nginx/default.conf:/etc/nginx/conf.d/default.conf
    depends_on: [api]
```

### 生产环境

```
Nginx (80) → Vue 3 静态文件
           → /api/* → Spring Boot (8080, 多实例)
           → /judge/* → 判题沙箱 (8081)
           → /ai/* → AI Gateway (8082)
           → /ws → WebSocket (8080)
           → /static/* → MinIO (CDN 回源)
```

## 十、技术决策记录

| 决策 | 选择 | 备选 | 理由 |
|---|---|---|---|
| 前端框架 | Vue 3 | React | 团队熟悉度、生态成熟 |
| 后端框架 | Spring Boot 3 | Go (Gin) | 微服务支持好、人才多 |
| sb3 解析 | Java 库 | Node.js | 与后端同语言，部署简单 |
| 判题沙箱 | Node.js + Docker | Java | scratch-vm 只有 JS 版 |
| AI Gateway | Python + FastAPI | Java | LLM SDK 生态最好 |
| 数据库 | MySQL + MongoDB | 纯 MySQL | 大 JSON 文档用 MongoDB 更合适 |
| 文件存储 | MinIO | 七牛/OSS | 开源、S3 兼容、自建可控 |
| 搜索 | Elasticsearch | MySQL FULLTEXT | 搜索能力远超 MySQL |
