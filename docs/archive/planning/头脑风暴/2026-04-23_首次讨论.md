# 💡 2026-04-23 首次讨论

> 参与者：用户 + AI 助手
> 主题：Scratch 社区平台立项 — 竞品分析 + 核心设计

---

## 讨论背景

用户希望基于 teaching-open 项目（v2.8）重新设计一个更强大的 Scratch 编程社区，融合在线编程、分享、刷题、竞赛、积分及 AI 功能。

## 竞品分析结论

分析了编程猫、小码王、斯坦星球、少儿编程 OJ 系统、Scratch 官方社区等平台后，得出关键发现：

1. **市场空白**：没有一个平台把「创作 + 社区 + 刷题 + 竞赛 + 积分 + AI」真正融合
2. **功能割裂**：社区平台不做 OJ，OJ 系统不做社区，两者完全分离
3. **AI 缺失**：现有平台几乎没有原生 AI 能力
4. **积分体系薄弱**：只有斯坦星球有简单的积分系统

## 核心创意点

### 🎯 创作与评测一体化
- 不是两个系统拼接，而是一个平台内的自然流程
- 学生创作 → 提交 → 自动评测 → 分享 → 获得积分，全在一个地方完成

### 🤖 AI 原生设计
- AI 不是后加的功能，而是从架构层面就为 AI 设计
- AI 编程助手：自然语言 → Scratch 积木
- AI 作品点评：自动分析作品的创意性/代码质量/复杂度
- AI 出题：根据知识点自动生成题目 + 测试用例
- AI 学习路径：根据学生画像个性化推荐

### 🔌 插件化架构
- 核心平台 + 插件机制
- 新语言（如 Lua、JavaScript）通过插件添加
- 新题型通过配置注册
- 新积木通过扩展 API 添加
- 硬件扩展（Micro:bit、Arduino）通过插件支持

### 🏆 完整激励体系
- 积分：多维度获取（创作/刷题/竞赛/社交/每日任务）
- 等级：8 级体系，从"代码新手"到"编程之神"
- 徽章：多类别多稀有度（创作/刷题/竞赛/社交/特殊）
- 称号：可展示的个性化称号
- 排行榜：周/月/总，多维度（热度/创作量/刷题量）

### ⚔️ 多样化竞赛
- 定时赛：经典 OJ 赛制
- 创意赛：给主题，提交作品，社区投票
- 组队赛：团队协作
- 段位赛：1v1 实时对战（像下棋一样匹配对手）

### 🔄 Remix 机制
- 借鉴 Scratch 官方的 Remix 概念
- 任何公开作品都可以被 Remix
- 自动记录作品族谱（原创 → Remix 链）
- 鼓励学习和二次创作

## 技术决策

| 决策 | 选择 | 理由 |
|---|---|---|
| 前端框架 | Vue 3 + TypeScript + Vite | 生态成熟、团队熟悉 |
| 后端框架 | Spring Boot 3 (主力) + Go (判题引擎) | Java 生态 + Go 高性能 |
| 数据库 | MySQL + MongoDB + Redis + ES | 各取所长 |
| 判题方案 | 混合方案（VM Headless + AI 评分） | 覆盖算法题和创意题 |
| 部署 | Docker + K8s | 微服务弹性伸缩 |
| 基础代码 | 复用 teaching-open 的用户/权限/课程模块 | 节省开发时间 |
